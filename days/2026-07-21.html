制作に使えそうな技術メモProduction notes

今日の実装メモは、preflight を boolean で終わらせず、shader が読める連続値にすることです。R=failed / unsupported risk、G=capability pass、B=queue / encode pressure、A=plane validity として持つと、7/13 以降の semantic / latency / selection / SOG retry の RGBA 設計にそのまま接続できます。JS 側では history pivot、queue、capability を指数減衰で管理し、GLSL は可視化に集中させます。実装を進めるなら、まず MediaRecorder.isTypeSupported で複数 mimeType の結果を capability array にし、canvas captureStream の有無、WebCodecs の有無、devicePixelRatio / canvas size / frame budget を加点減点して uniform 化します。

Today's production note is to treat preflight as continuous shader-readable values, not a final boolean. R = failed / unsupported risk, G = capability pass, B = queue / encode pressure, and A = plane validity connects directly to the RGBA state design used since 07-13 for semantic, latency, selection, and SOG retry. Keep history pivot, queue, and capability as JS-side exponential decay values; let GLSL focus on visualization. Next, use MediaRecorder.isTypeSupported across multiple mimeTypes, then fold in canvas captureStream availability, WebCodecs availability, devicePixelRatio, canvas size, and frame budget as uniform scores.

The recent direction is consistent: splat/shader work is becoming an instrument for reading production state, not just a visual surface. The sequence from 07-13 semantic mask, 07-14 latency residue, 07-15 selection boolean, 07-16 temporal diff, 07-17 SOG retry budget, 07-18 fog/audio scope, 07-19 adaptive upload gesture, 07-20 temporal hysteresis, and 07-21 export preflight plane expands state from runtime into editing and export. A strong mapping is SuperSplat selection = R, PlayCanvas LOD quality = G, Babylon retry/cooldown = B, stream validity = A, Web Audio = temporal gain, gesture = seed, texture upload = pressure, history pointer = hysteresis, Orient plane = grid basis, and video preflight = capture risk.
