注目事例と何が面白いかNotable examples and why they matter

Babylon.js 9.14.0 の HtmlTexture と GS streaming LOD、PlayCanvas の GSplat AA / lodRange 系、SuperSplat のブラウザ編集環境は、いずれも『高品質な面や点群を、どこまでリアルタイムに触れるか』という同じ問題に向かっています。面白いのは、フォトリアルな splat をそのまま見せるだけでなく、読みやすい HTML 面、SDF 的なシャープな記号、粗い周辺視の粒度を混ぜられる点です。つまり、現実スキャンの質感と情報デザインの輪郭を同じ空間の中で LOD 制御できるようになります。

Babylon.js HtmlTexture plus GS streaming LOD, PlayCanvas GSplat AA and lodRange work, and the SuperSplat browser editor all point to the same question: how far can high-quality surfaces and splats remain interactive in real time? The interesting part is not merely showing photoreal splats, but mixing them with readable HTML surfaces, sharp SDF-like symbols, and coarse peripheral grains. This makes it possible to control scanned texture and information-design edges with one spatial LOD rule.

表現・インタラクションのアイデアExpression and interaction ideas

短期案は『スコープ型 LOD』です。ユーザーのポインタ、視線、手の位置の近くでは SDF 文字や輪郭を細くし、周辺ではガウス粒を大きくして曖昧に残します。中期案は、昨日の depth gate tunnel にこのスコープを重ね、ゲートを通過するたびに情報面だけがシャープになり、背景 splat は粗く流れる構成です。長期案は、SuperSplat で作ったスキャン空間に HTMLTexture / HtmlTexture を混ぜ、現実由来の場に『焦点を合わせたときだけ読める注釈』を置くことです。

The short-term idea is scope-based LOD: near the pointer, gaze, or hand position, SDF glyphs and outlines become thin and sharp, while the periphery remains as larger Gaussian grains. The mid-term idea is to layer this scope onto yesterday's depth-gate tunnel so that only information surfaces sharpen when passing through a gate while background splats smear coarsely. The long-term idea is to combine SuperSplat scans with HTMLTexture or HtmlTexture and place annotations in real-world spaces that become readable only on focus.

制作に使えそうな技術メモProduction notes

SDF/MSDF の輪郭品質は fwidth だけでなく、実際に表示される UV 拡大率や距離に合わせて幅を変えると破綻が減ります。Gaussian Splatting 側は splat の個数だけでなく、ピッキング、LOD、ソート、ストリーミング、XR stereo 表示が制作上の詰まりどころです。HTML-in-Canvas 系は drawElementImage / texElementImage2D / copyElementImageToTexture のように『DOM を画像化して GPU 側へ渡す』発想なので、CSS の読みやすさと WebGL のポスト処理を両立できる一方、更新頻度とヒットテスト設計を最初から分けて考える必要があります。

For SDF/MSDF edges, adapting width to visible UV magnification and distance can reduce artifacts beyond a naive fwidth approach. For Gaussian Splatting, the production bottlenecks are not only splat count but also picking, LOD, sorting, streaming, and XR stereo rendering. HTML-in-Canvas APIs such as drawElementImage, texElementImage2D, and copyElementImageToTexture follow the pattern of snapshotting DOM into GPU-side textures, enabling readable CSS and WebGL post-processing together, but update frequency and hit testing should be designed separately from the start.

俯瞰メモBird's-eye memo

最近の傾向は、HTML/DOM を 3D に貼る話と、Gaussian Splatting をリアルタイム制作に入れる話が近づいていることです。組み合わせ候補としては、1. HTMLTexture + foveated shader、2. SuperSplat scan + SDF 注釈、3. GSplat LOD + Web Audio、4. depth gate + XR stereo splat が有望です。育てる価値が高いテーマは『情報の読みやすさを常時最大にしない』設計で、見る・近づく・通過する・音が立つときだけ解像度が上がる表現です。

The recent trend is convergence between rendering HTML/DOM into 3D and bringing Gaussian Splatting into real-time production. Promising combinations are: HTMLTexture plus foveated shaders, SuperSplat scans plus SDF annotations, GSplat LOD plus Web Audio, and depth gates plus XR stereo splats. The high-value theme is avoiding always-maximal readability: resolution rises only when looking, approaching, passing through, or when sound events occur.
