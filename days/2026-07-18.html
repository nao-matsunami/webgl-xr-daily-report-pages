表現・インタラクションのアイデアExpression and interaction ideas

今日の中心案は『LOD Fog Audio Scope』です。7/17 の current budget lens / predicted lens / retry cooldown / failed risk を残しつつ、retry を霧、stale prediction を残光、音反応を瞬間的な visibility impulse として扱います。ポインタを動かすと budget lens が tile を精細化し、押下すると Web Audio 的な impulse が stale residue と fog density を一時的に増幅します。短期では音声入力なしの synthetic impulse で十分です。中長期では実 Web Audio analyser の low/mid/high band を、A=stream validity、B=latency/cooldown、G=LOD quality、R=selection keep に掛け合わせます。

The core idea is LOD Fog Audio Scope. It keeps the 07-17 current budget lens, predicted lens, retry cooldown, and failed risk, but treats retry as fog, stale prediction as afterglow, and audio response as a visibility impulse. Moving the pointer sharpens tiles through the budget lens; pressing injects a Web-Audio-like impulse that temporarily amplifies stale residue and fog density. In the short term, a synthetic impulse is enough. In the mid/long term, real Web Audio analyser low/mid/high bands can modulate A = stream validity, B = latency/cooldown, G = LOD quality, and R = selection keep.

制作に使えそうな技術メモProduction notes

実装メモとしては、重い postprocess ではなく fragment shader 内で tile state、fog、scan ring をまとめて描くのが速いです。実データへ進める場合、SOG chunk ごとの状態 texture を RGBA で持ち、R=selection/current keep、G=LOD quality、B=retry cooldown または latency、A=validity にします。PlayCanvas / SuperSplat の Streamed SOG は 1M Gaussians 以上で LOD bundle 化され、lowest LOD から表示し、device ごとの Gaussian budget に収める設計なので、shader 側では『現在見せている精細度』と『本当は見せたい精細度』の差を可視化すると価値があります。three.js r185 の HTMLTexture は annotation source、Babylon の GaussianSplattingStream retry は cooldown source、Trionn 型の Web Audio は impulse source として接続できます。

Implementation note: keep it lightweight by drawing tile state, fog, and scan rings in one fragment shader instead of using heavy post-processing. With real data, store per-SOG-chunk state in RGBA: R = selection/current keep, G = LOD quality, B = retry cooldown or latency, A = validity. PlayCanvas / SuperSplat Streamed SOG turns splats above 1M Gaussians into LOD bundles, displays from the lowest LOD, and stays within device-dependent Gaussian budgets, so the useful shader signal is the difference between the detail currently shown and the detail desired. three.js r185 HTMLTexture can be the annotation source, Babylon GaussianSplattingStream retry can be the cooldown source, and Trionn-style Web Audio can be the impulse source.
