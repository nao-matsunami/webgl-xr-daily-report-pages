表現・インタラクションのアイデアExpression and interaction ideas

今日の中心案は「Temporal Selection Diff Scope」です。cyan は今の keep lens、amber は前回または予測された keep lens、violet は両者の差分 residue、red は cut / stream risk、yellow は depth annotation gate。ポインタを動かすと現在選択が追従し、押下中は stale residue と risk が強くなります。作品としては、鑑賞者の視線や手の動きが、空間に青い現在形と琥珀色の過去形を残し、その差分だけが紫に燃える。制作ツールとしては、selection operation、semantic confidence、SOG validity、latency residue を同じ画面で見られるため、splat scene の編集ログをビジュアルに検証できます。

The core idea is Temporal Selection Diff Scope. Cyan is the current keep lens, amber is the previous or predicted keep lens, violet is the diff residue between them, red is cut / stream risk, and yellow is the depth annotation gate. Moving the pointer updates the live selection; holding amplifies stale residue and risk. As an artwork, the viewer's gaze or hand motion leaves a blue present and an amber past in space, with only the difference burning violet. As a production tool, selection operation, semantic confidence, SOG validity, and latency residue become visible on the same surface, making the edit log of a splat scene visually inspectable.

制作に使えそうな技術メモProduction notes

試作は依存なしの HTML + WebGL + GLSL です。fragment shader 内で procedural noise から semantic / stale field を作り、現在 lens と過去 lens の差分を abs(current - previous) として residue に変換しています。実制作では、R=selection keep、G=semantic confidence、B=latency/residue、A=stream validity の継続規約に加え、operation mode(add/subtract/intersect)と frame age を別 texture か packed channel に持たせるのが良いです。PlayCanvas 側では SOG unload/reload/cache event を A または age channel へ、SuperSplat 側では active selection operation と numeric volume を R/operation channel へ、Babylon/three.js 側では MSDF/HTMLTexture annotation を depth gate として重ねる構成が現実的です。

The prototype is dependency-free HTML + WebGL + GLSL. The fragment shader generates semantic and stale fields from procedural noise, then converts the difference between current and previous lenses into residue via abs(current - previous). In production, keep the running convention R = selection keep, G = semantic confidence, B = latency/residue, A = stream validity, and add operation mode (add/subtract/intersect) plus frame age as either another texture or packed channels. A practical setup is: PlayCanvas SOG unload/reload/cache events feed A or age, SuperSplat active operation and numeric volume feed R/operation, and Babylon/three.js MSDF/HTMLTexture annotation becomes the depth gate.
