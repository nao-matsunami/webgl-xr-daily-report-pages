今日の重要トピック

three.js r185 の HTMLTexture 対応、Babylon.js 9.14.0 の HtmlTexture、PlayCanvas v2.20.0 の copyElementImageToTexture と stereo XR gsplat が並び、HTML/DOM 系の情報面を 3D 空間に持ち込む下地が複数エンジンで揃ってきています。

注目事例と何が面白いか

model-viewer の multi-model scene と Canvas/Video texture の例は、3D 本体と情報レイヤーを一つの空間で扱う設計の参考になります。特に『固定HUDではなく、意味レイヤーを空間に浮かせる』方向へつなぎやすいです。

表現・インタラクションのアイデア

常時読ませる UI ではなく、注目時だけ焦点面を合わせて情報が浮き上がる見せ方が有効です。普段は空間の気配として薄く存在し、視線やポインタ、会話の瞬間だけ読める層に切り替える構成が自然です。

制作に使えそうな技術メモ

HTML-in-Canvas を前提にしておけば、Canvas2D や DOM ベースで組んだ意味レイヤーを、後から WebGL texture / postprocess / XR 投影へ拡張しやすいです。三次元側で全部を書き切るより、情報面の管理がかなり軽くなります。
