PlayCanvas Engine v2.20.6 fixes crashes around gsplat component detach, GPU device hangs when picking gsplats on Windows/NVIDIA, SOG asset unloads while still referenced by the unified world, mid-load unload behavior in SOG bundle parsing, and destroyed textures returning from the loader cache. It also stores markup text colors as linear-space vertex palettes, tightening both color handling and asset lifecycle behavior.

注目事例と何が面白いかNotable example and why it matters

今日の面白さは、cache の不正な再利用や unload 中 asset の扱いを、単なるバグ修正ではなく visual state として見られる点です。streaming された SOG/GSplat は、読み込み済み、選択中、破棄待ち、再ロード中、stale などの状態を持ちます。これを shader 側で heatmap、scanline、border、stale red として描くと、デバッグ UI と作品表現の境目が近づきます。

The interesting point is that cache reuse and unloading behavior can become visual state, not just bug-fix territory. A streamed SOG/GSplat asset can be loaded, selected, pending disposal, reloading, or stale. Mapping those states to shader heatmaps, scanlines, borders, and stale red highlights brings debugging UI closer to visual expression.

制作に使えそうな技術メモProduction notes

実装では、asset cache key と GPU texture handle の lifecycle を分けて考えるのが重要です。node cache は再利用できても、texture が destroyed なら render path へ戻してはいけません。表現側では、cache slot ごとに active、age、stale、reloadPulse を持ち、linear palette で色を作ると WebGPU/WebGL 間の見え方を揃えやすくなります。

Implementation-wise, separate the asset cache key lifecycle from the GPU texture handle lifecycle. A node cache entry may be reusable, but a destroyed texture must not re-enter the render path. For expression, keep active, age, stale, and reloadPulse per cache slot, and derive color through a linear palette to keep WebGPU/WebGL appearance more consistent.
