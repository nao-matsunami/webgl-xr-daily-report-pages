注目事例と何が面白いかNotable examples and why they matter

面白いのは、HTML-in-Canvas が単なる UI 表示ではなく、読みやすい HTML/CSS を shader の入力素材にする提案として見えてきたことです。three.js の HTMLTexture 更新はその受け皿で、Babylon.js の sRGB 修正は情報 texture を post-process に通す時の色の信頼性に効きます。PlayCanvas の GSplat streaming 修正は地味ですが、scan や splat asset を切り替える展示表現では unload まわりの安定性がそのまま体験品質になります。

The interesting point is that HTML-in-Canvas reads less like simple UI rendering and more like a proposal for turning readable HTML/CSS into shader input. The three.js HTMLTexture update is one receiver for that direction, while Babylon.js's sRGB fix matters for color reliability when information textures go through post-processing. The PlayCanvas GSplat streaming fix is quiet, but unload stability directly affects experience quality in exhibitions that swap scan or splat assets.

表現・インタラクションのアイデアExpression and interaction ideas

短期では、前回の『屈折する情報面』に SDF edge と疑似 Gaussian LOD を足し、読める領域をポインタ/視線/手の周辺だけに限定するのがよさそうです。中期では、HTMLTexture 由来の注釈膜を実 GSplat scene の上に重ね、近い場所だけ文字、輪郭、粒子密度が同期して立ち上がる表現にできます。長期では、XR 空間で現実スキャンの上に『普段は環境に溶けているが、焦点を合わせると読める』注釈レイヤーを育てる価値があります。

In the short term, add SDF edges and pseudo Gaussian LOD to the previous refractive information surface, limiting readable regions to the pointer, gaze, or hand neighborhood. In the mid term, layer HTMLTexture-derived annotation membranes over a real GSplat scene so text, outlines, and particle density rise together near the viewer. In the long term, this can become an XR annotation layer over real-world scans: normally blended into the environment, readable when focused.

制作に使えそうな技術メモProduction notes

実装上は、情報 texture と splat/scan を同じ focus 値で制御すると設計しやすいです。focus を 0..1 の場として持ち、SDF の antialias 幅、splat の cell size、屈折量、色収差、ポストプロセス強度へ分配します。HTML-in-Canvas を使う場合は、DOM snapshot 更新頻度、texElementImage2D / copyElementImageToTexture の転送コスト、DOM 側の hit testing と描画位置の同期を別々に測る必要があります。Babylon.js 9.15.0 の sRGB 修正は、こうした情報 texture を WebGPU pipeline へ載せる時の色管理の確認ポイントになります。

A practical implementation pattern is to drive both information textures and splat/scan rendering with the same focus value. Keep focus as a 0..1 field and distribute it to SDF antialias width, splat cell size, refraction amount, chromatic shift, and post-process intensity. With HTML-in-Canvas, measure DOM snapshot update rate, texElementImage2D / copyElementImageToTexture transfer cost, and synchronization between DOM hit testing and drawn position separately. Babylon.js 9.15.0's sRGB fix is a useful checkpoint for color management when these information textures enter a WebGPU pipeline.

俯瞰メモBird's-eye memo

最近の流れは、派手な新機能よりも『読める情報を GPU 素材として破綻なく扱う』方向にまとまっています。HTML-in-Canvas は情報の素材化、Babylon.js の sRGB/tree-shaking 修正は色と依存関係の安定化、PlayCanvas の GSplat streaming 修正は重い空間 asset の差し替え安定化です。組み合わせ候補は、1. HTMLTexture + SDF focus、2. Gaussian LOD + readable membrane、3. GSplat streaming + 音反応、4. WebXR Layers + shader controlled annotations。育てる価値が高いテーマは『読める/読めないの境界をインタラクションにする』ことです。

The recent direction is less about flashy new features and more about handling readable information as GPU material without breaking. HTML-in-Canvas materializes information; Babylon.js sRGB/tree-shaking fixes stabilize color and dependencies; PlayCanvas GSplat streaming fixes stabilize heavy spatial-asset swaps. Promising combinations are: HTMLTexture plus SDF focus, Gaussian LOD plus readable membranes, GSplat streaming plus audio reactivity, and WebXR Layers plus shader-controlled annotations. The high-value theme remains making the boundary between readable and unreadable interactive.
