注目事例と何が面白いかNotable examples and why they matter

注目は PlayCanvas / SuperSplat の Streamed SOG 説明です。大きな splat は 1M Gaussians 以上で LOD bundle 化され、低い LOD から near-instant に表示し、desktop / mobile / XR ごとの Gaussian budget に合わせて detail を上げ下げします。ここで面白いのは、LOD が作品の裏側ではなく、表現の時間軸になることです。近づくと detail が上がるだけでなく、retry 中の tile、cooldown 中の tile、validity が落ちた tile、前フレームの予測とずれた tile を見せれば、巨大な記憶空間が「読み込まれながら存在する」演出になります。

The notable example is PlayCanvas / SuperSplat's Streamed SOG documentation. Large splats become LOD bundles at and above 1M Gaussians, display quickly from a low LOD, then raise or lower detail under desktop / mobile / XR Gaussian budgets. The interesting point is that LOD is not just an implementation detail; it can become an expressive timeline. If the scene shows tiles that are retrying, cooling down, losing validity, or diverging from previous prediction, a huge memory space can appear as something that exists while being loaded.

表現・インタラクションのアイデアExpression and interaction ideas

今日の中心案は「SOG Streaming Retry Budget Scope」です。ポインタ位置を current budget lens、時間で動く円を predicted budget lens とし、その差分を magenta の stale diff にします。grid tile は仮想 SOG chunk、green は valid / high LOD、amber は cooldown retry、red は failed stream risk。押下中は drag/drop reload pressure として retry と failed risk が増えます。作品としては、観客が空間へ手を伸ばすと、記憶の粒子が green に精細化し、追いつかない部分だけ amber / red に発熱する。制作ツールとしては、selection history と LOD retry を同じ目で見られます。

The core idea is SOG Streaming Retry Budget Scope. The pointer is the current budget lens; the moving time-based circle is the predicted budget lens; their difference becomes magenta stale diff. Grid tiles act as virtual SOG chunks: green is valid / high LOD, amber is cooldown retry, and red is failed stream risk. Holding the pointer simulates drag/drop reload pressure and increases retry plus failure risk. As an artwork, reaching into the space sharpens memory particles in green while lagging areas heat up amber or red. As a tool, selection history and LOD retry can be inspected through one visual vocabulary.

制作に使えそうな技術メモProduction notes

試作は依存なしの HTML + WebGL + GLSL です。実データでは、R=selection/current keep、G=semantic confidence または LOD quality、B=latency / stale diff、A=stream validity という前回までの channel 規約を維持しつつ、tile ごとの retryCooldown と lastSuccessfulLOD を別 texture に持たせると扱いやすいです。Babylon.js 9.17.0 の deferred LOD switch retry は B または cooldown texture、safe-orbit limits は current lens の移動可能範囲、PlayCanvas SOG unload/reload/cache event は A channel、SuperSplat folder D&D は reloadPressure へ接続できます。three.js / Babylon の depth-aware MSDF/HTMLTexture は、どの tile が高 LOD に達しているかをラベル表示する gate に向きます。

The prototype is dependency-free HTML + WebGL + GLSL. With real data, keep the channel convention from previous reports: R = selection/current keep, G = semantic confidence or LOD quality, B = latency / stale diff, A = stream validity. Add retryCooldown and lastSuccessfulLOD per tile as another texture. Babylon.js 9.17.0's deferred LOD switch retry maps to B or a cooldown texture; safe-orbit limits map to the allowed movement range of the current lens; PlayCanvas SOG unload/reload/cache events map to A; SuperSplat folder drag-and-drop maps to reloadPressure. three.js / Babylon depth-aware MSDF or HTMLTexture is a good gate for labeling which tiles actually reached high LOD.
