注目事例と何が面白いかNotable examples and why they matter

今日の注目は PlayCanvas v2.21.0 と SuperSplat v2.32.2 の組み合わせです。片方は glTF KHR_gaussian_splatting / SPZ / volumetric fog を engine 側に入れ、もう片方は 8K 書き出しと muxer finalize の待ちを editor/export 側で詰めています。面白いのは、splat が『標準フォーマットとして読めるか』と『高解像度で最後まで撮れるか』が同時に制作体験の品質になっている点です。これを UI の成功/失敗表示だけにせず、fog、grid、chunk heat、pivot lag として空間内に出すと、巨大な splat 空間の準備状態そのものが作品になります。

The notable combination is PlayCanvas v2.21.0 plus SuperSplat v2.32.2. One side adds glTF KHR_gaussian_splatting, SPZ, and volumetric fog at the engine layer; the other tightens 8K export and muxer-finalization waits at the editor/export layer. The interesting point is that whether a splat can be read as a standard format and whether it can be captured at high resolution both become part of production quality. If those states are shown as fog, grids, chunk heat, and pivot lag instead of just UI success/failure, the readiness of a huge splat space becomes the piece.

The central idea is Export Resolution glTF Scope. 制作に使えそうな技術メモProduction notes

実装メモは、昨日の preflight を codec 可否だけで終わらせず、format gate と resolution pressure を分離することです。R=8K/export failure risk、G=KHR_gaussian_splatting/SPZ compatibility、B=muxer finalize wait または encode queue、A=working-plane / chunk validity と置くと、7/13 以降の RGBA state 設計にそのまま接続できます。PlayCanvas v2.21.0 の WebGL2 view uniform buffer は、camera/view ごとの状態をまとめて渡す発想に近く、wide line renderer は診断 grid や chunk 境界に向きます。volumetric fog はエラーの赤表示より柔らかく、chunk が読めていない/まだ finalize していない状態を空間の密度として見せやすいです。

The production note is to split yesterday's preflight into format gate and resolution pressure rather than only codec availability. R = 8K/export failure risk, G = KHR_gaussian_splatting/SPZ compatibility, B = muxer-finalization wait or encode queue, and A = working-plane / chunk validity connects directly to the RGBA state design used since 07-13. PlayCanvas v2.21.0's WebGL2 view uniform buffers align with passing per-camera/view state in bundles, while the wide line renderer is useful for diagnostic grids and chunk boundaries. Volumetric fog is softer than red error overlays and can present unread chunks or unfinished finalization as spatial density.

俯瞰メモBird's-eye memo

最近の流れは、selection / semantic / SOG retry / upload / export preflight がすべて『状態を shader で読む』方向に収束しています。今日の PlayCanvas v2.21.0 で glTF Gaussian と SPZ が engine に入り、SuperSplat v2.32.2 で 8K export の完了待ちが明示されたので、runtime と authoring/export の境界がかなり近づきました。組み合わせ候補としては、KHR_gaussian_splatting の fallback point cloud を low LOD 表現、SPZ/SOG を high LOD 表現、volumetric fog を未確定状態、HTMLTexture/MSDF を depth-aware label にする構成が強いです。育てる価値が高いテーマは『標準化されつつある splat asset を、読み込み・編集・書き出しの時間差ごと表現する scope』です。

The recent trend is converging on reading selection, semantic state, SOG retry, upload, and export preflight through shaders. With PlayCanvas v2.21.0 bringing glTF Gaussian and SPZ into the engine and SuperSplat v2.32.2 making 8K export waiting explicit, runtime and authoring/export are moving closer together. A strong combination is KHR_gaussian_splatting fallback point clouds as low LOD, SPZ/SOG as high LOD, volumetric fog as unresolved state, and HTMLTexture/MSDF as depth-aware labels. The theme worth growing is a scope for standardized splat assets that expresses loading, editing, and exporting delays as part of the visual language.
